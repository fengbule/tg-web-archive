# 最终验收报告

本报告只记录脱敏后的结果，不包含服务器地址、账号、手机号、频道名称、Telegram API 凭据或 Session。

## 服务器与部署

- Debian 服务器使用 Docker Compose 部署。
- 应用和 Nginx 容器均配置 `restart: unless-stopped`。
- 数据库、视频、临时文件、Session 和缩略图均持久化到 `/data/telegram-video-manager`。
- 服务器和容器重启后，数据库、现有 Telegram 授权和封面缓存均保留。
- 健康检查返回 HTTP 200。

## Web 与安全

- 管理员登录、退出和未授权 API 拦截通过。
- 未登录访问媒体接口返回 HTTP 401。
- 文件路径和重命名目录穿越测试返回 HTTP 400。
- 管理员密码使用 PBKDF2 或兼容 bcrypt 哈希，不明文保存。
- Telegram API 凭据加密保存；验证码和两步验证密码不保存。
- 查看 API ID/API Hash 必须在已登录后台内再次验证管理员密码；错误密码返回 403，响应禁止缓存，测试日志不包含凭据内容。
- 管理员密码修改后旧 Cookie 全部失效，旧密码登录失败，新密码登录成功；新哈希和会话版本持久化到数据库。
- 独立查看密码的设置、修改、错误密码拦截、旧密码失效和恢复使用管理员密码均通过。
- 运维脚本提供忘记管理员密码时的安全重置入口，Shell 语法检查通过；重置不会删除媒体、数据库或 Telegram Session。
- 源码、Git 工作树和容器日志均未检出实际服务器密码、管理员密码、API Hash 或 Session 内容。

## 下载、封面和播放

- 双 worker 下载队列、任务判重、磁盘保护和 SQLite 锁竞争恢复通过。
- 真实视频下载到部分进度后主动重启容器，临时文件未归零，启动后从已有偏移续传并最终完成。
- 真实 1.6GB 下载在 326,107,136 字节处通过网页暂停；暂停期间文件大小稳定不变，点击继续后从相同位置增长。
- 手动暂停、继续、取消使用不同状态；暂停保留 `.part`，只有取消才删除临时文件。
- 网络异常状态、自动重试状态、手动重试保留临时文件和用户主动取消删除临时文件的逻辑已覆盖。
- 真实 Telegram MP4 下载完成并写入媒体库。
- Telegram 原封面下载与本地缓存通过。
- FFmpeg 媒体库封面、时长和分辨率读取通过。
- Chrome 原生播放器成功读取视频元数据。
- HTTP Range 返回 206，拖动播放进度通过。
- FastAPI 鉴权后通过 Nginx internal/X-Accel-Redirect 直接传输媒体；未登录返回 401，内部媒体路径直访返回 404。
- 10MB Range 服务器端读取从约 0.088 秒降至 0.032 秒；浏览器远距离快进到 590、1200、3000 秒均在 10 秒内恢复可播放。
- 发现并按 Telegram 原文件字节修复历史验收期间产生的 78 个不一致分块；整部 55 分钟视频完整解码退出码 0、错误 0。
- 真实完成文件大小与 Telegram 声明大小一致；下载和播放链路均未调用视频转码。
- 电脑端原画播放器宽度 1180px，浏览器读取到的 1276×720 分辨率和 3315 秒时长与原文件一致。
- 不兼容编码会显示中文提示，并保留原文件下载入口。
